Dynamic Book Reader

Unfortunately, I couldn't keep the code but I recorded the functionality below!

Responsive Design

To address accessiblility issues with the previous book reader, I coded a new book reader with a responsive design. There are three different layouts: portrait with one page, portrait with two pages, and landscape.

Highlight Feature

Created a highlighting feature that displayed popups for word clicks and multiple word selections using AngularJS factories and modal services. Utilized a custom API service for persistent highlighting data.

Drawing Feature

Coded a drawing feature that supported drawing, highlighting and stamping. Each of the drawing tools has their own sub-toolbar with the color/stamp options for each tool. This used the another custom API service for data persistence.

Listening Feature

Made a listening feature for the book reader that reads the page or the entire book. When reading each word it highlights the phrases and words as they are being read so students can follow along.

Note Feature

Implemented a feature that can record notes for each page in the book. It supports a draggable note icon and can be edited by clicking on either the icon or the button. Has custom API service for data persistence.

Journal Feature

This feature has two areas: word journal and reflections. The each with their own custom component they handle saving new entries, editing entries, and deleting entries. This also uses a custom API service for persistent data storage.
